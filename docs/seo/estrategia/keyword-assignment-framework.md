# Keyword Assignment Framework

## Objetivo
Este documento define cómo se asignan las keywords dentro de Yatezzitos Global para evitar canibalización, aprovechar inventario real y asegurar que cada búsqueda transaccional relevante tenga una URL dueña dentro del sitio.

---

## Regla principal
Cada keyword principal debe tener una sola URL dueña.

No se permite repetir la misma keyword principal en dos URLs distintas.

---

## Lógica general de asignación

### 1. Página madre de ciudad
La mejor keyword transaccional del destino debe asignarse a la página principal de ciudad.

### 2. Fichas de yates
Las demás keywords del destino se reparten entre las fichas de producto, siempre que:
- tengan intención de compra,
- se relacionen con el tipo real de embarcación,
- y no repitan otra keyword ya asignada.

### 3. Páginas por tipo de embarcación
Solo deben crearse si la ciudad tiene inventario suficiente.

### 4. Blog
Las keywords menos transaccionales o de apoyo pueden moverse al blog.

---

## Prioridad de asignación

La prioridad para decidir a qué URL se asigna una keyword es:

1. intención transaccional
2. volumen
3. facilidad de posicionar
4. relación con inventario real
5. lógica comercial del sitio

---

## Reglas por tipo de URL

### Página de ciudad
Debe quedarse con:
- la keyword más fuerte,
- más transaccional,
- más representativa del destino.

### Página de producto
Debe quedarse con:
- keywords específicas,
- variantes transaccionales,
- búsquedas relacionadas con ese tipo real de embarcación.

### Página por tipo de embarcación
Debe quedarse con:
- keywords del tipo “catamarán en ciudad”
- “velero en ciudad”
- “lancha en ciudad”
cuando exista inventario que lo justifique.

### Blog
Debe quedarse con:
- variantes informativas,
- apoyo semántico,
- guías,
- playas,
- preguntas frecuentes,
- contenido que ayude al enlazado interno.

---

## Relación entre ciudad y producto
Una página de ciudad no debe intentar capturar todas las keywords del mercado.

Su papel es capturar la keyword madre del destino.

Las fichas de producto deben absorber el resto de las keywords transaccionales disponibles, siempre que:
- no haya duplicación,
- no exista mejor URL dueña,
- y la keyword tenga sentido con el producto real.

---

## Reglas de compatibilidad semántica
La keyword asignada a una ficha debe tener sentido con la embarcación real.

### Ejemplos
- una keyword de catamarán no debe asignarse a un yate si existe un catamarán real en ese destino
- una keyword de pesca debe asignarse a una embarcación o página que realmente represente pesca
- una keyword de lujo debe apoyar una ficha coherente con esa intención

---

## Regla de ciudad con inventario mínimo
Se puede trabajar con fuerza el SEO de una ciudad si existe al menos una embarcación activa.

---

## Regla para páginas por tipo de embarcación
Solo se justifican si en esa ciudad existe inventario mínimo representativo.

Regla actual:
- un yate
- un catamarán
- un velero

---

## Scoring recomendado
Para priorizar keywords, se recomienda una fórmula simple:

### Score SEO sugerido
- volumen: 40%
- facilidad de posicionar: 30%
- intención transaccional: 20%
- ajuste al inventario real: 10%

### Nota
Si una keyword tiene volumen muy alto, puede seguir siendo prioritaria aunque sea más difícil, especialmente en páginas madre de ciudad.

---

## Excepción importante
Si dos keywords tienen volumen similar, debe preferirse la más fácil de posicionar.

---

## Política de no duplicación
Antes de asignar una keyword nueva, siempre debe revisarse que:

- no exista ya otra URL dueña,
- no esté ya usada como keyword principal,
- no canibalice una página madre,
- no duplique otra ficha.

---

## Tabla maestra recomendada
La gestión debe apoyarse en una hoja con columnas como:

- ciudad
- URL
- keyword
- búsquedas
- intención
- prioridad
- estado

---

## Estado recomendado
El campo “estado” puede usarse con valores simples como:
- pendiente
- asignada
- optimizada
- publicada
- revisada

---

## Flujo recomendado de trabajo
1. elegir ciudad
2. detectar keyword madre
3. asignar keyword a página de ciudad
4. repartir keywords restantes entre productos
5. detectar oportunidades de categoría
6. mandar informativas al blog
7. revisar que no exista duplicación
8. optimizar on-page
9. enlazar internamente
10. medir resultados

---

## Regla operativa final
No se crea una página ni se asigna una keyword solo porque exista volumen.

La keyword debe:
- tener intención útil,
- tener una URL lógica,
- respetar la estructura del sitio,
- y ayudar a generar reservas.

---

## Conclusión
La asignación de keywords en Yatezzitos debe ser manual, estratégica y controlada.

El objetivo no es repartir keywords al azar.  
El objetivo es construir una arquitectura donde cada búsqueda importante tenga una URL correcta, única y orientada a ventas.
