# Estado actual del CRM

## Resumen
Actualmente, el centro operativo del CRM de Yatezzitos está en **GoHighLevel**.

Ahí vive la mayor parte de la operación comercial relacionada con:
- captación de leads,
- seguimiento comercial,
- automatizaciones,
- formularios,
- correos,
- WhatsApp,
- SMS,
- y parte del flujo de cotización y reserva.

Aunque ya existe una base funcional, el sistema todavía necesita más orden, mejor segmentación, mejor estructura de datos y una relación más clara entre clientes, propietarios, socios comerciales y procesos internos.

## Rol actual de GoHighLevel
GoHighLevel funciona hoy como el núcleo operativo del negocio para:

- almacenar contactos,
- centralizar información comercial,
- activar automatizaciones,
- enviar mensajes,
- recibir formularios,
- coordinar parte del seguimiento de ventas,
- guardar datos relevantes del cliente.

También se utiliza como soporte para procesos relacionados con:
- cotizaciones,
- reservas,
- recibos,
- seguimiento antes y después del viaje.

## Qué información vive hoy en el CRM
Actualmente, en GoHighLevel vive principalmente la información relacionada con:

### Clientes turistas
- nombre,
- correo,
- teléfono,
- datos de contacto,
- información relacionada con su viaje,
- seguimiento comercial,
- estatus de cotización o reserva,
- parte de la comunicación histórica.

### Procesos comerciales
- cotización enviada,
- avance en el proceso de reserva,
- pagos o anticipos,
- seguimiento manual o automatizado,
- interacción en distintos canales.

### Automatizaciones
- envío de correos,
- envío de SMS,
- envío de WhatsApp,
- formularios integrados,
- webhooks,
- secuencias parciales de seguimiento.

## Integración actual entre web y CRM
La web actual en WordPress sí tiene conexión con el CRM, pero no de forma completamente ordenada todavía.

Actualmente existen integraciones como:

- formularios conectados a GoHighLevel,
- webhooks que envían información,
- páginas dinámicas que leen datos de contactos,
- páginas personalizadas para cotización, gracias y reserva,
- integración con OTP mediante Twilio en el buscador principal.

También existe una integración realizada para que WordPress pueda leer datos del CRM y mostrarlos en páginas personalizadas mediante tokens o parámetros.

## Qué sí está automatizado hoy
Actualmente, sí existen automatizaciones útiles en el CRM, especialmente en procesos como:

- envío de cotización,
- envío de parte de la información de reserva,
- flujo relacionado con el anticipo del 50%,
- uso de formularios para disparar acciones,
- envío de información al cliente cuando el equipo llena ciertos datos.

## Qué sigue siendo manual hoy
A pesar de que ya existe automatización parcial, todavía hay una fuerte dependencia del trabajo manual del equipo.

Hoy siguen siendo manuales o parcialmente manuales cosas como:

- llenado de encuestas internas para disparar cotizaciones,
- llenado de formularios internos para generar recibos o reservas,
- parte del seguimiento operativo,
- organización de propietarios,
- organización de embarcaciones,
- validación de datos,
- captura de nueva oferta,
- parte del control de pagos y confirmaciones.

## Problemas actuales del CRM
Hoy el CRM tiene valor real, pero también presenta varios problemas estructurales:

### 1. Desorden en la base de datos
Clientes, propietarios y otros perfiles no siempre están estructurados de forma ideal.

### 2. Falta de separación clara de perfiles
Aún no existe una estructura suficientemente limpia para distinguir y operar correctamente entre:
- clientes turistas,
- propietarios,
- brokers,
- administradores,
- capitanes,
- agencias,
- aliados.

### 3. Dependencia de procesos manuales
Muchos procesos importantes todavía requieren que alguien del equipo llene formularios o haga tareas repetitivas.

### 4. Integración incompleta con WordPress
Aunque ya existe conexión entre WordPress y GoHighLevel, todavía no está resuelta de forma sólida para:
- cuentas de usuario,
- disponibilidad,
- onboarding de propietarios,
- estructura ordenada de embarcaciones,
- operación más avanzada.

### 5. Falta de orden en la oferta
La información de propietarios, socios comerciales y embarcaciones no vive todavía en una estructura suficientemente limpia y centralizada.

## Estado actual de propietarios dentro del CRM
Se intentó trabajar información de propietarios dentro de la misma cuenta del CRM donde también viven los clientes, pero el resultado no quedó bien organizado.

Esto provocó:
- mezcla de perfiles,
- desorden,
- dificultad para segmentar,
- poca claridad operativa.

Actualmente, el alta de propietarios y embarcaciones todavía depende demasiado de procesos externos, formularios poco optimizados y apoyo manual.

## Canales conectados o relacionados con la operación
Hoy el CRM participa o recibe información relacionada con canales como:

- sitio web,
- formularios web,
- WhatsApp,
- correo,
- SMS,
- Instagram,
- Facebook,
- leads provenientes de redes y otros canales.

## Flujo comercial actual resumido
El flujo actual del CRM puede resumirse así:

1. llega el lead,
2. se captura información,
3. se genera cotización,
4. se solicita anticipo del 50%,
5. se envía información de reserva o recibo,
6. se da seguimiento comercial u operativo,
7. parte del proceso continúa manualmente.

## Qué está funcionando bien
Actualmente sí están funcionando estas bases:

- GoHighLevel como núcleo operativo,
- automatizaciones parciales útiles,
- formularios y secuencias conectadas,
- flujo comercial funcional,
- integración básica entre CRM y sitio,
- almacenamiento de contactos,
- seguimiento de clientes reales.

## Qué necesita mejorar con urgencia
Las prioridades de mejora del CRM son:

- ordenar mejor la base de datos,
- separar correctamente perfiles,
- automatizar más pasos del embudo,
- organizar propietarios y embarcaciones,
- mejorar secuencias y seguimiento,
- reducir trabajo manual,
- mejorar conexión entre CRM y sitio web,
- preparar el sistema para disponibilidad en tiempo real y futura web app.

## Decisión estratégica actual
GoHighLevel no se va a reemplazar de inmediato.

La estrategia correcta es:
- documentar mejor el estado actual,
- ordenarlo,
- estructurarlo mejor,
- automatizar lo que falta,
- y usarlo como base operativa mientras la arquitectura evoluciona.

## Rol del CRM en la siguiente etapa
En la siguiente etapa del proyecto, el CRM debe convertirse en una operación mucho más clara para tres grupos:

### 1. Clientes
Para vender mejor, dar seguimiento y acompañar la experiencia.

### 2. Socios comerciales
Para captar y ordenar propietarios, agencias, brokers y oferta disponible.

### 3. Equipo interno
Para operar mejor, reducir errores y aumentar productividad.

## Conclusión
El CRM actual de Yatezzitos ya sostiene una parte importante del negocio, pero necesita orden estructural, mejor segmentación, más automatización y mejor integración con la web para convertirse en una verdadera base escalable para crecimiento nacional e internacional.
